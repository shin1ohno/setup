# コミュニケーション・スタイル

このリポジトリでの文章生成（チャット応答、Notion ページ、README、議事録など）に共通する規律。

## 日本語出力

- ですます調を維持、常体との混在禁止
- hedge 禁止: 「思います」「たぶん」「〜かもしれません」「検討する価値があります」「確認しますか？」を使わない。不確実性は数値か条件で書く（例: 「8 割確度で X」「A の場合 Y、B の場合 Z」）
- 形容詞・副詞は具体数値で置換: 「大幅改善」→「800ms → 200ms」、「ほぼ完了」→「10 のうち 9 完了」、「軽微」→「ファイル 2 本、追加 18 行」
- 後送り禁止: 「後ほどお知らせします」「次回確認できます」ではなく今やる
- 人名は「さん」付け（Slack `@-mention` を除く）

## Pyramid Principle（文書作成）

- 結論を先頭に置く（BLUF: Bottom Line Up Front）
- 根拠を MECE にグループ化、各段落の冒頭文がその段落の結論
- 階層は 3 レベル以内
- 散文を既定、bullet は本当に補助になる時だけ
- Marginal utility test: 削っても価値が変わらない文は削る

## 一次情報の扱い

- Notion / Slack の内容は本文に複製せず、リンクと 1〜2 行の要約に留める
- 複製は腐敗源（原本が更新されたとき、コピー側が古くなる）
- リンクには「いつ時点の内容か」を明記すると寿命が延びる

## 5+ 案件のグループ化

- 列挙が 5 件以上になるときは、ユーザー目的のテーマでグループ化する
- ファイル単位 / 重要度単位ではなく、「読み手が何を判断するか」でグループ化する
- グループをそのまま AskUserQuestion の選択肢にできるとよい
