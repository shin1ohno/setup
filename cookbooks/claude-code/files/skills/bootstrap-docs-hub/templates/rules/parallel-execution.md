# 並列実行とサブエージェント

## 基本原則

- 独立したタスクは Agent ツールの並列呼び出し（同一メッセージ内に複数 tool_use）で同時実行する
- 1 エージェント = 1 タスク。複数役割を 1 エージェントに負わせない
- 長時間の調査・検索（>60 秒見込み）は `run_in_background: true` で会話を止めない

## 会話開始時の背景投入

- ユーザーの最初のメッセージが非自明なタスクの場合、関連する一次情報（Notion / Slack の関連ページ・スレッド）は背景で先に取得しておく
- 取得結果が来る前に主たる作業を始める。idle 待機しない

## 適用例

- Notion 検索と Slack 検索が両方必要 → 同一メッセージで両方の MCP ツールを呼ぶ
- 複数の Notion ページを読む → fetch を並列で呼ぶ
- 既存ファイルを複数読む → Read を並列で呼ぶ

## 直列にすべきケース

- 後段のツール呼び出しに必要な値が、前段の出力にしかない場合
- 同一ファイルへの編集（先後関係を保証するため）
